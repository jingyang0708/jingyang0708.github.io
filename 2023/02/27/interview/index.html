接收数据

    第二种方式（限制类型）：

            },

作用域和执行上下文的区别？

1、函数的执行上下文是在函数调用的时候生成的，而其作用域是在创建的时候就已经创建好了
2、作用域包含了若干个函数执行上下文，也有可能是零个（因为函数还没有执行）

this的指向问题

this的指向只有在调用时才能确定，因为this是执行上下文的一部分
1、全局作用域中的函数：this的指向是window
2、对象内部的函数：this的指向是其对象本身
3、构造函数：this的指向是他的实例对象
4、由apply、call、bind改造的函数：this的指向是他的第一个参数
5、箭头函数：箭头函数没有自己的this,他的this是其上下文中的this

如何理解同步和异步

同步：同步就是按照代码的顺序，从上往下依次执行
异步：可以理解为是一种并行行为，意思就是可以不必等到上一段代码执行完毕，就可以执行其他的代码。
js之所以需要异步是因为js是单线程的，常用的异步场景有：定时器，ajax请求、事件绑定等
补充：
进程和线程
进程：就是程序的运行环境（厂房）
线程：就是实际进行运算的一个东西（工人）

同步和异步
同步：通常情况下代码都是自上向下一行一行执行的
前边的代码不执行，那么后面的也不执行
同步的代码执行会出现阻塞的情况
一行代码执行慢，会影响到整个代码的执行

解决同步的问题：
java python :通过多线程解决
node.js:通过异步方式解决

异步：一段代码的执行不会影响其他的代码
异步的代码不能通过return设置返回值
特点：
1、不会阻塞其他代码的执行
2、需要通过回调函数来返回结果
基于回调函数的异步带来的问题
1、代码的可读性差
2、可调试性差
解决问题：Promise（存储异步调用的返回值）
就是一个可以用来存储数据的对象
存储数据的方式比较特殊，这种特殊方式使得Promise可以用来存储异步调用的数据

实现异步的方式有哪些

1、回调函数
2、事件监听
3、发布订阅模式
4、Promise
5、async/await
6、利用生成器实现

怎样理解Promise对象

理解

Promise：可以帮助我们解决异步中的回调函数的问题
Promise就是一个用来存储数据的容器，他拥有一套特殊的存取数据的方式，这个方法可以使得他里面可以存储异步调用的数据（结果）

创建Promise

//创建Promise时，构造函数中需要一个函数作为参数（function(){} 和箭头函数都可以）
//Promise构造函数的回调函数，他会在创建Promise的时候就调用，调用时会有两个参数传递进去
const promise = new Promise((resolve, reject) => {
  //resolve,reject时两个函数，通过这两个函数，可以向Promise里面存储数据
  //resolve在执行正常的时候存储数据，reject在执行错误时存储数据
  // reject('bkpp')
  //通过函数来向Promise中添加数据，好处就是可以用来添加异步调用的数据
  setTimeout(() => {
    resolve('pp')
  }, 1000)

读取数据

/*
从Promise中读取数据
  通过Promise的实例方法then来读取Promise中存储的数据
  then方法里面需要两个回调函数来作为参数，回调函数用来获取Promise中存储的数据
      通过resolve存储的数据，会调用第一个函数返回,
            我们可以在第一个函数里面编写处理函数的代码
      通过reject存储的数据或者出现异常时，会调用第二个函数返回
          我们可以在第二个函数里面编写处理异常的代码

*/
promise.then((result) => {
  console.log('promise中的数据', result);
}, (reason) => {
  console.log('promise中的数据', reason);
})

Promise中维护了两个隐藏的属性

 PromiseResult 用来存储数据
 PromiseState 用来记录Promise的状态（三种）
       pending (进行中)
       fulfilled（完成）通过resolve存储数据
       rejected (拒接，出错了) 出错了或者通过reject存储数据
  PromiseState只能修改一次，修改后永远不会在变   pending--->fulfilled  pending--->rejected

流程：
   当Promise创建时，Promise的初始值为pending
      当通过resolve存储数据时， PromiseState变为 fulfilled（完成）
          PromiseResult变为存储的数据
      当通过reject存储数据或者出错时， PromiseState变为  rejected (拒接，出错了) 
          PromiseResult变为存储的数据 或者异常对象

    当我们通过then读取数据的时候，相当于为Promise设置了回调函数
      如果PromiseState变为fulfilled（完成），则调用then的第一个回调函数来返回结果
       如果PromiseState变为 rejected (拒接，出错了)，则调用then的第二个函数来返回结果

Promise中的方法

promise中 then() catch() finally() 都会返回一个新的Promise
Promise中会存储回调函数的返回值 (finally除外)
finally的返回值，不会存储到Promise中

then()

对Promise进行链式调用的时候，后面的方法读取的时前面方法的返回值

catch()

当Promise出现异常时，而整个调用链中没有catch，那么就会向外抛出异常
catch()我们一般写在最后面，永远都是后面的解决前面的问题，catch()不能发现自身的异常，自能发现前面的异常

Promise 中的catch
catch()是用来处理异常的数据，当我们的数据中没有异常的数据或者错误数据，就会跳过这个catch()直接执行下一行代码
当数据中有异常的数据，那么then()就会被跳过，而去执行catch()方法

静态方法

静态方法（就是直接通过类调用的方法）
Promise.resolve(10) 创建一个立即完成的Promise
Promise.reject(‘错误！！！’) 创建一个立即拒绝的Promise
Promise.all([…]) 同时返回多个Promise的执行结果,其中有一个错，就返回错误，返回的数组里面就是得到的数据
Promise.allSettled([…]),同时返回多个Promise的执行结果,无论成功或失败，都会返回，返回的数组里面是对象型的
{status: ‘fulfilled’, value: 3}
{status: ‘rejected’, reason: 10}
Promise.race([…]) 返回执行最快的Promise,不考虑对错，都返回
Promise.any([…]) 返回完成最快的Promise,只有当所有的都为reject的情况下才报错

Promise对象有以下两个特点：

1、对象的状态不受外界的影响。Promise对象一共有三种状态，pending、fulfilled、rejected。状态值只会被异步的结果决定，其他操作无法改变。
2、状态一旦改变，就不会再变，且任何时候都可以得到这个结果。状态值会由pending–>fulfilled或pending–>rejected，这时既是resloved

Promise有以下三个缺点：

1、Promise一旦执行就无法取消
2、不可设置回调函数，其内部的错误无法捕获
3、当Promise处于pending的时候，无法得知具体发展道路那一步

cooKies机制和session机制的区别

1、cookies数据保存在客户端，session数据保存在服务端
2、cookies可以减轻服务器的压力，但是不安全，容易进行cookies欺骗
3、session安全一点，但是占用服务器资源

get和post的区别

1、url可见性
get:url参数可见，post：url参数不可见
2、数据传输上
get:通过拼接url进行传递参数
post:通过body体传递参数
3、缓存性
get请求是可以缓存的，post请求不可以缓存
4、后退页面的反应
get请求页面后退时，不产生影响
post请求页面后退时，会重新提交请求
5、传输的数据大小
get一般传输的数据大小不超过2k-4k
post请求传输的数据大小根据php.ini配置文件设定，也可以无限大
6、安全性
原则上post肯定要比get请求安全，毕竟传输参数时url不可见，但是有人闲的没事会抓包，所以也没啥区别
7、数据包
简单来说，get产生一个TCP数据包，post生成两个TCP数据包，对于get方式的请求，浏览器会把http header和data一并发送出去，然后服务器响应200（返回数据）；对于post来说，浏览器会先发送header,服务器响应100 continue,浏览器在发送data,服务器响应200（返回数据）

textContent,innerText,innerHTML,value的区别

1、textContent用来获取和设置文本内容，与innerText不同的是textContent可以获取到的内容包括了元素中的style标签和script标签的内容
2、innerText只能获取和设置文本内容。不能设置html标签
3、innerHTML可以获取和设置html标签
4、value，用来获取表单数据

什么叫DOM事件流

事件发生时会在元素节点之间按照特定的顺序传播，整个过程分为捕获阶段、目标阶段和冒泡阶段，这个传播过程就是DOM事件流
事件冒泡：就是从小到大，事件源到document
事件捕获：就是从大到小，document到事件源

如何让事件先冒泡后捕获

在原本的的事件流中，是先捕获在冒泡
可以给目标元素，绑定两个事件监听器，一个用于冒泡，一个用于捕获，然后将两个事件的执行顺序是按照代码的顺序执行的，所以先写冒泡，在写捕获，让弄胡就会按照这个歌顺序执行
如果是对于非目标元素，给捕获的事件加上一个定时器，这时捕获事件就成为了异步任务，进入了宏任务队列中，这样做的目的就是让冒泡事件执行完在执行捕获事件。

js的执行机制

JS是单线程的，他的运行基于事件循环(event loop) 

调用栈
   栈
      栈是一种数据结构，后进先出
   调用栈中，放的是要执行的代码 

任务队列
  队列
     队列是一种数据结构，先进先出
  任务队列里面的是将要执行的代码
  当调用栈中的代码执行完成后，任务队列中的代码才会按照顺序依次进入到栈中执行
  在JS中任务队列有两种
    一种是宏任务队列：大部分代码都去宏任务队列中排队
    另一种是微任务队列：promise的回调函数（then.catch finally）他们就回去微任务队列中排队

整个流程：
   1、执行调用栈里面的代码
   2、执行微任务队列里面所有的任务，
   3、当微任务队列中的所有执行完毕之后，在去执行宏任务队列中的

HttP协议

网络基础
网络的服务是基于 请求和相应的

 https:// lilichao.com /hello/index.html   
  https:// 协议名
  lilichao.com 域名 整个网络中存在着无数个服务器， 每一个服务器都有他自己的唯一标识 ，
         这个标识被称为ip地址 192.168.1.17.但是ip地址不方便记忆，域名就相当于ip地址

  /hello/index.html 网站资源的路径

当在浏览器中输入地址后发生了什么

 https:// lilichao.com /hello/index.html         
  a、DNS解析，获取网站的ip地址
     通过网址去查找服务器的ip地址  （DNS相当于电话本，ip地址相当于电话号）
  b、浏览器需要和服务器建立连接（tcp/ip） (通过三次握手建立连接)
  c、向服务器发送请求 (HttP协议)
  d、服务器处理请求，并返回响应 （http协议）
  e、浏览器将相应的页面进行渲染
  f、断开和服务器的连接 （四次挥手）

  建立是客户端建立，断开是谁发数据谁断开（一般是服务器断开的多）

客服端如何和服务器建立（断开）连接

   通过三次握手（建立）和四次挥手（断开）
三次握手（建立连接）
     是客服端和服务器建立连接的过程
       1、客服端向服务器发送连接请求
           SYN
       2、服务器收到连接请求，向客户端返回消息
          ACK（同意） SYN (申请和客户端连接)  
       3、客户端向服务器发送同意连接的请求
          ACK（同意）
四次挥手 （断开连接）
      1、客户端向服务器发送请求，向服务器发送数据完毕，请求断开连接
           FIN
      2、服务器笑傲那个客户端返回数据，知道了
          ACK
      3、服务器向客户端返回数据，数据收完了，可以断开连接
          FIN  ACK
      4、客户端向服务器发送数据，我知道了，可以断开
           ACK

     请求和相应实质上就是一段数据，只是这段数据要遵循一个特殊的格式
        这个特殊的格式就是由HTTP协议决定 （http协议就是规定请求和相应的协议的）

TCP/IP 协议族

TCP/IP 协议族 中包含了一组协议
这组协议规定了互联网中所有的通信的细节 （http协议就是这里面的一个）

网络通信的过程 由四层组成

       发数据 从上到下
       1、应用层
           软件的层面，浏览器，服务器都属于应用层
       2、传输层（负责拆包，把大文件拆成一个一个的小包，丢包不会有大的影响）
           负责对数据进行拆分，把大数据拆分成一个一个的小包
       3、网络层 （）
             负责给数据盖戳，添加信息
       4、数据链路层
             负责传输信息

      HttP协议就是应用层的协议，用来规定客户端和服务器之间通信的报文格式

服务器的主要功能就是

       1、可以接收到浏览器发送的请求报文
       2、可以向浏览器返回响应报文

报文是什么？（message）

浏览器和服务器之间的通信是基于请求和相应的
 浏览器向服务器发送请求（request）
  服务器向浏览器返回相应（response）
    浏览器向服务器发送请求，相当于浏览器给服务器写信
    服务器向浏览器返回相应，相当于服务器给浏览器回信       
    这个信在HTTP协议中就称为报文，而HTTP协议就是对这个报文进行规定的
请求报文（request）
          客户端发给浏览器的的报文成为请求报文
          请求报文的格式如下：
             请求首行
             请求头
             空行（分割的作用）
             请求体



        请求首行：就是请求报文的第一行
           GET    /06-http%E5%8D%8F%E8%AE%AE/01-http%E5%8D%8F%E8%AE%AE.html?username=%E9%9D%99%E9%A6%99  HTTP/1.1
            第一部分 GET 表示请求的方式  get表示发送的是get请求
              现在常用的方式就是get和post请求
              get请求用来向服务器请求资源
              post请求主要用来向服务器发送数据

           第二部分
           /06-http%E5%8D%8F%E8%AE%AE/01-http%E5%8D%8F%E8%AE%AE.html?username=%E9%9D%99%E9%A6%99   
             表示请求资源的路径
               ？后面的内容叫查询字符串
               查询字符串是一个名值对结构，一个名字对应一个值
                使用 = 连接，多个名值对之间用 &分割   username=静香&password=1234
               get请求通过查询字符串将数据发送给服务器
                  由于查询字符串会在浏览器地址栏中显示
                  所以他安全性较差
                  get请求没有请求体，同时由于url地址长度有限，所以get请求无法发送较大的数据

              post请求通过请求体来发送数据
                在浏览器中通过载荷来查看
                post请求通过请求体发送数据，无法在地址栏直接查看，所以安全性较好
                请求体的大小没有限制，可以奉送任意大小的数据

              如果需要向服务器发送请求，尽量用post

           第三部分 HTTP/1.1
             表示http协议的版本


        请求头：名值对结构，给服务器看的，用来告诉服务器我们浏览器的信息
            每一个请求头都有他的作用
             Accept；浏览器可以接受的文件类型
             Accept-Encoding ：浏览器允许的压缩的编码
             Accept-Language：客户端可以接受的语言
              User-Agent：用户代理 （用来描述浏览器信息的字符串）

        空行：用来分割请求头和请求体


        请求体
          post通过请求体来发送数据

     响应首行
          HTTP/1.1 405 Method Not Allowed
          HTTP/1.1 HttP协议版本
           405 表示响应状态码
           Method Not Allowed 表示对响应状态码的描述

           响应状态码的规则
             1xxx 请求处理中
             2xxx 表示成功
             3xxx 表示请求的重定向
             4xxx 表示客户端错误
             5xxx 表示服务器错误

     响应头
       也是一个一个的名值对结构，用来告诉浏览器响应的信息
         Content-Type：用来描述响应体的类型
           Content-Length：用来描述响应体的大小
